## ‚Ěó The Problem

You work across multiple AI orchestrators ‚ÄĒ **Claude Code**, **Gemini CLI**, **Codex**, **OpenClaw**, **Hermes**.

Each one is its own universe.

```
You build an agent in Gemini CLI.
You open Claude Code.
It knows nothing about it.

You configure an MCP in Claude Code.
You open Codex.
Gone.

You write a skill, a command, a cron job ‚ÄĒ in one tool.
Every other tool: blank slate.
```

There is no shared layer. Every orchestrator hoards what lives inside it.  
You end up **copy-pasting configs, duplicating agent definitions, and maintaining the same thing in five places** ‚ÄĒ and the moment one changes, everything else is out of date.

---

## ‚ú® What WaslGenie Does

WaslGenie installs itself as a **native skill inside each orchestrator** and runs as an **external CLI daemon** alongside it.

When sync is triggered ‚ÄĒ manually or automatically ‚ÄĒ WaslGenie:

1. **Scans** the known config directories of every supported orchestrator on your machine  
   (`~/.claude/`, `~/.gemini/`, `~/.codex/`, `~/.openclaw/`, `~/.hermes/`)
2. **Discovers** all agents, MCPs, skills, commands, and cron jobs ‚ÄĒ wherever they were originally created
3. **Writes a minimal stub file** into every other tool's equivalent directory ‚ÄĒ not a copy, not a duplicate ‚ÄĒ just enough for the native tool to load the original
4. **The original file never moves.** The tool that created it owns it forever.

> No file copying. No format conversion. No duplication.  
> Just cross-references that let each tool load what the other built.

---

## ūüĒ¨ How Cross-Referencing Works

Say you create an agent inside Gemini CLI:

```
~/.gemini/agents/researcher.md   ‚Üź original, owned by Gemini
```

After `waslgenie sync`, WaslGenie writes a minimal stub into every other tool:

```
~/.claude/agents/researcher.md   ‚Üź stub, written by WaslGenie
~/.codex/agents/researcher.md    ‚Üź stub, written by WaslGenie
~/.openclaw/agents/researcher.md ‚Üź stub, written by WaslGenie
```

Each stub contains only the minimum that native tool needs to load the original:

```markdown
---
# researcher
waslgenie_ref: ~/.gemini/agents/researcher.md
origin: gemini
---
Refer to source definition at ~/.gemini/agents/researcher.md
```

Claude Code reads its stub ‚Üí loads the Gemini original ‚Üí agent is live.  
**Zero bytes duplicated. Zero maintenance.**

The same pattern applies across every asset type:

```
~/.gemini/agents/       ‚Üí  stubs written to  .claude  .codex  .openclaw  .hermes
~/.claude/mcp/          ‚Üí  stubs written to  .gemini  .codex  .openclaw  .hermes
~/.codex/skills/        ‚Üí  stubs written to  .claude  .gemini  .openclaw  .hermes
~/.openclaw/commands/   ‚Üí  stubs written to  .claude  .gemini  .codex  .hermes
~/.hermes/crons/        ‚Üí  stubs written to  .claude  .gemini  .codex  .openclaw
```

**Source of truth = the tool that created it first. Always. Forever.**

| Asset | Scanned From | Synced To |
|---|---|---|
| **Agents / Sub-agents** | `~/.{tool}/agents/` | All other tools' agent dirs |
| **MCP Servers** | `~/.{tool}/mcp/` | All other tools' MCP configs |
| **Skills / Prompts** | `~/.{tool}/skills/` | All other tools' skill dirs |
| **Custom Commands** | `~/.{tool}/commands/` | All other tools' command dirs |
| **Cron Jobs** | `~/.{tool}/crons/` | All other tools' cron dirs |

## ūüöÄ Installation

WaslGenie is cross-platform via `npx` ‚ÄĒ no global install required:

```bash
npx wasl-genie install
```

This detects every supported orchestrator on your machine and registers WaslGenie as a native skill inside each one.

**Or install globally:**

```bash
npm install -g wasl-genie
waslgenie install
```

## ūüóÉÔłŹ Registry Storage

WaslGenie keeps its own state separately from all orchestrators. You choose the scope at install time:

**User-level** (default ‚ÄĒ available across all your projects):
```
~/.waslgenie/
‚Ēú‚ĒÄ‚ĒÄ registry.json     ‚Üź every discovered asset + origin tool + stub locations
‚Ēú‚ĒÄ‚ĒÄ stubs/            ‚Üź log of every stub written and when
‚ĒĒ‚ĒÄ‚ĒÄ config.json       ‚Üź your scope and preferences
```

**Workspace-level** (scoped to current project only):
```
.waslgenie/
‚Ēú‚ĒÄ‚ĒÄ registry.json
‚Ēú‚ĒÄ‚ĒÄ stubs/
‚ĒĒ‚ĒÄ‚ĒÄ config.json
```

Switch anytime:
```bash
waslgenie config --scope workspace
waslgenie config --scope user
```
