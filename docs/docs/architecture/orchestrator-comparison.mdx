# Provider Native Surfaces and Sync Mapping

WaslaGenie is a portable AI workspace layer. It does not impose one provider's folder structure on another provider. The syncer discovers logical assets and each adapter reads or writes the destination's native representation.

## Portable Assets

| Wasla asset | Meaning | Current behavior |
|-------------|---------|------------------|
| `agent` | Custom agent persona and instructions | Mirrored to native custom-agent files where the provider supports them |
| `skill` | On-demand reusable capability with `SKILL.md` resources | Mirrored as native Agent Skills directories |
| `mcp` | Named MCP server configuration | Normalized internally and converted to native JSON layout |
| `context` | Project-wide instruction document | Mirrored to the provider's project instruction entry point |

Rules and file-scoped instructions are not modeled as agents or skills. They need a future `instruction` asset contract before they can be synchronized without losing meaning.

OpenClaw remains experimental until its adapter contract is validated against authoritative provider documentation.

## Adapter Responsibilities

### Paths

An adapter maps one logical asset to native paths. For example, one portable `reviewer` agent can be materialized as:

```text
.claude/agents/reviewer.md
.gemini/agents/reviewer.md
.opencode/agents/reviewer.md
.cursor/agents/reviewer.md
.github/agents/reviewer.agent.md
```

### MCP Shape Conversion

Claude, Gemini, Cursor, and Copilot CLI accept `mcpServers` entries shaped around `command` and `args`. OpenCode and VS Code Copilot require conversion at the adapter boundary:

```text
portable { command: "npx", args: ["server"] }
  -> OpenCode { type: "local", command: ["npx", "server"], enabled: true }
  -> VS Code { type: "stdio", command: "npx", args: ["server"] }
```

The scanner applies the inverse conversion when those native files are the source of a sync.

### Agent Content

Markdown bodies and common YAML fields such as `name` and `description` are portable across the current file-backed agent providers. Provider-specific tool names, permissions, delegation settings, and model options are not currently normalized. An adapter must not claim that those fields have been translated until an explicit schema converter exists.

## Sync Lifecycle

1. Scan installed providers at their native paths.
2. Group matching logical assets by `name` and `type`.
3. Select the most recently edited version for each asset.
4. Write that version through each destination adapter.
5. If a previously synchronized copy is deleted and remaining copies were unchanged, propagate the deletion.
6. If another copy was edited after synchronization, its edit wins over a deletion and repopulates destinations.

`watch` runs the same sync behavior when native asset paths change. Writes that already match their destination are skipped so a synchronized write does not continually trigger itself.

## Current Boundary

Supported now: native paths for agents and skills, MCP JSON shape conversion for OpenCode and VS Code Copilot, newest-edit resolution, deletion propagation, and watch over adapter-native paths.

Not yet modeled as independent portable assets: scoped instruction collections such as `.github/instructions/*.instructions.md` or `.cursor/rules/*.mdc`, and semantic rewriting of provider-specific custom-agent frontmatter.
